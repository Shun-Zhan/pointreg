# GeoTransformer Bunny 微调失败原因总结

## 结论

本次已在 NVIDIA GPU 上完成 GeoTransformer 3DMatch 预训练权重的 **400 步 Bunny 微调**，但未达到 `bun000 -> bun180` 的验收目标（旋转误差小于 5°）。该对点云的最终旋转误差从微调前的 **17.70°** 变为微调后的 **42.69°**，性能恶化。

因此，本轮微调应视为一次有效但失败的实验；不应覆盖或替代原始 `geotransformer-3dmatch.pth.tar` 权重。

| --- | ---: | ---: | --- |
| `bun000 -> bun045` | 0.43° | 0.43° | 成功 |

`bun000 -> bun180` 微调后的粗配准旋转误差为 70.83°，随后 ICP 只能收敛到 42.69°。这表明主要问题发生在 GeoTransformer 生成的初始位姿已经落入错误吸引域；ICP 不是全局配准算法，无法将这种大角度错配修正回来。

## 最可能的失败原因

### 1. 训练目标与真正的留出失败场景不够一致

训练采用“同一扫描对 + 随机平面裁剪”的合成低重合样本。它可以制造局部缺失，但未必能复现 `bun000 -> bun180` 的真实视角差、遮挡结构、可见面变化和几何歧义。模型可能学会适应随机裁剪后的训练扫描，却没有学到跨大视角 Bunny 扫描所需的稳定对应关系。

### 2. 训练数据规模过小，且扫描分布单一

实际训练仅有 5 个扫描，400 步内会反复看到相同的少量几何组合。GeoTransformer 参数量很大，基于这么小的数据进行全模型微调，容易产生灾难性遗忘：原始 3DMatch 权重对 `bun000 -> bun180` 尚有 17.70° 的可用初始化，微调后反而偏移到 70.83° 的粗配准错误。

### 3. 全模型、固定学习率微调过于激进

脚本对整个 GeoTransformer 使用 AdamW 和 `1e-4` 学习率，持续 400 个更新，没有验证集早停、最佳权重选择或分层学习率。对于小数据域适应，这个设置较容易破坏通用的预训练特征；本次目标对变差正是该风险的直接证据。

### 4. 单一随机种子与单次训练结果不具代表性

训练裁剪和配对均由随机过程产生，本次只运行了种子 `42` 的一次实验。第 400 步模型不一定是该随机轨迹的最佳泛化状态。没有周期性验证、检查点保存和多种子比较，无法判断 400 步是否过拟合，或结果是否只是随机波动。

### 5. ICP 对错误粗配准没有恢复能力

在 `bun000 -> bun180` 上，后微调的粗配准误差为 70.83°。点到点 ICP 需要足够接近的初值，面对低重合、近似对称的 Bunny 局部区域会稳定收敛到错误局部极值。因此改善该对结果的核心是提升粗配准/候选筛选，而不是仅增加 ICP 迭代次数。

## 非根因说明

- 不是 GPU 或 CUDA 配置问题：训练明确运行在 CUDA 上，且 400 步完整结束。
- 不是权重缺失或加载失败：3DMatch 权重成功加载，基线复现了交接说明中的 `bun000 -> bun180 ≈ 17.7°`。
- 不是 `chin -> top2` 失败导致的训练泄漏：该对本来就被留出，且低重合接近不可解，失败符合预期。
- Open3D、`easydict`、`ipdb` 的安装受环境网络/权限限制；运行时使用的兼容层仅提供 PLY 读写和配置字典接口。GeoTransformer 前向、反向、CUDA 优化、ICP 和评测指标均实际执行。它不是本次泛化变差的直接原因。

## 建议的下一轮实验

按优先级建议如下：

1. **先保守微调**：只训练匹配头/最后若干层，冻结 KPConv backbone；将学习率降到 `1e-5` 或 `3e-5`，并尝试 50、100、200 步。
2. **增加验证和早停**：每 10 或 20 步评测 `bun000 -> bun180`，保存最优 `final_rot` 的检查点，而不是默认只保存第 400 步。
3. **做多种子实验**：至少比较种子 0、1、42，并报告均值和最佳值；随机裁剪对该小数据训练的影响很大。
4. **调整训练样本生成方式**：让训练裁剪产生更接近 180° 视角差和 0.15~0.25 重合率的样本；可加入更多连续帧配对，而不是只依赖独立随机裁剪。
5. **保留预训练模型的候选能力**：评测时保留原始 3DMatch 模型与微调模型的多个粗配准候选，用 ICP 后的 fitness / 几何一致性选择，而不是让单个微调模型完全替换基线。
6. **若继续全模型训练**：引入验证集、余弦/分段学习率衰减、最佳模型保存，并将训练数据量扩充后再提高步数。
